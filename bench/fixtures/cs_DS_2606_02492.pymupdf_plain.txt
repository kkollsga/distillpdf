Abstract
Parameterized complexity has always been concerned with practical computing: by confining
combinatorial explosion to a secondary parameter k, one can uncover why and how many NP-hard
problems are effectively tackled in practice. Today, however, the scale of data has changed: scientists
study Big Data, which is so large that even quadratic dependence in the total input size n is
unaffordable. Therefore, what constitutes a practical algorithm has also changed. Classically,
parameterized complexity is blind to the difference between defining fixed parameter tractability
multiplicatively (i.e. f(k) · nc) or additively (i.e. f(k) + nc). But what if the constant c is one and
we require true linearity, is this distinction still inconsequential? Here, we define and explore Truly
Linear FPT (TLFPT) – that is O(n) + f(k) – and show that it is a strict subset of Linear FPT
(LFPT) – that is O(n) · f(k) – via diagonalization.
Populating TLFPT requires careful consideration of linear-time algorithmics and data structures.
We meet many inhabitants of TLFPT: SAT, Vertex Cover, Min-Max Matching, (n −k)-
Coloring, Diverse Pair of Matchings, k-Path, and H-Coloring. Our parameterizations are
equally varied. Beyond classical parameters like solution size, we leverage two parameters, treedepth
and BFS-width, which are particularly well-suited to the TLFPT regime. We do so by developing
techniques based on depth- and breadth-first search.
For parameterized complexity to be of service to the scientific community, we need to contend
with Big Data. For sufficiently large inputs, FPT beyond linear may not suffice. Thus, there is a
practical and theoretical need for more ambitious goals. TLFPT is a first step forward.

Introduction
There is a simple premise at the core of parameterized complexity: it is unrealistic to quantify
the running time of algorithms solely in terms of the total input size n. In many cases there
is a smaller, problem-relevant quantity k to which combinatorial explosion can be confined.
From this perspective, an efficient algorithm is one running in time f(k) · nc, where f is
a function independent of n and the constant c is independent of both n and k. This is
fixed-parameter tractability (FPT).
This guiding philosophy is far from a mere mathematical curiosity; it bears genuine
practical weight. Even for modest values of n and k, the difference between an nk running
time and a 2kn running time can span dozens of orders of magnitude [10]. In other words,
isolating the exponential dependence within k is not just elegant theory, but is often the only
hope for obtaining an algorithm that has guarantees of both correctness and a running time
that terminates within the remaining thermodynamic lifetime of the universe.
At this point, an observer might reasonably ask: if the goal is to design provably efficient
algorithms, why insist on a running time multiplicative in n of the form nc · f(k)? Would
the seemingly stronger bound nc + f(k) be even better, especially when n is expected to
dwarf k? This instinct is sound, but in the classical framing of parameterized complexity,
the distinction barely matters: the additive and multiplicative notions of FPT coincide up to
polynomial factors1 in n.
However, today inputs can be so large that even a single superlinear pass over one is
infeasible. Applications to giant instances were the subject of some discussion at the recent
Dagstuhl seminar “Recent Trends in Graph Decomposition” [28]), where the present paper’s
foundations were conceived. When linear-time processing is the limit of feasibility, there is a
natural question that classical parameterized complexity did not originally need to confront:
Forced to incur only linear dependence on n, do the additive and multiplicative
definitions of FPT remain equivalent?
Our answer is a definitive no: we draw a sharp distinction between linear FPT (LFPT)2
– algorithms running in time n · f(k) – and truly linear FPT (TLFPT) – algorithms running
in time O(n) + f(k). Furthermore, we show that TLFPT is a strict subset of LFPT, by
diagonalization, yielding the refined hierarchy
TLFPT ⊂LFPT ⊂FPT ⊆W[1] ⊆W[2] ⊆. . .
Populating TLFPT requires careful consideration of the assumed model of computation
(cf. [22]). As is standard in many fine-grained contexts, we work in a word-RAM model.
Furthermore, mirroring the classical FPT/kernelization equivalence, we show that being in
TLFPT corresponds exactly to admitting kernelization running in time O(n + k)3. This
insight lets us identify problems that are already known to be in TLFPT (e.g.
Max
Leaf Spanning Tree on weighted graphs [25] and d-Hitting Set on hypergraphs [1]).

TLFPT is useful: Big Data and Vertex Cover
We continue in the tradition of illustrating
the value of FPT with a concrete example using Vertex Cover, which asks, given a graph
G = (V, E) and integer k, whether there exist k vertices in the graph which together cover
all edges [8, 10, 11]. In those textbooks’ motivating examples, values of n ranged from 1000
to 10000, with of k between 10 and 70. In 2026, major scientific endeavors are supported by
the processing of petabytes of data [4]: for our example, we take n equal to a hundred billion
and k = 100. The state-of-the art algorithm for Vertex Cover [5] runs in TLFPT time5:
O(1.27k + |V | + |E|). Our example values of n and k make stark the distinction between
TLFPT and LFPT: an algorithm running in time 1.2738k · n would take 1021 operations
to terminate; an algorithm running in time 1.2738k + n would take only 1011 operations to
terminate. For an average laptop, this is the difference between minutes and millennia.

Technical preliminaries
Let [n] := {1, 2, . . . , n} and [i, j] := {i, i + 1, . . . , j}. Unless stated
otherwise graphs G = (V, E) are simple, loopless and undirected. The union of graphs (V, E1)
and (V, E2) is (V, E1 ∪E2). We refer the reader to Garey and Johnson [20] for the standard
definitions of a maximal matching and dominating set, and the corresponding decision
problems. Given a loopless graph H, an H-coloring of a graph G is a graph homomorphism
G →H. When H is the complete graph on n vertices, we speak of a proper n-coloring.
The H-coloring problem asks if a given graph G admits an H-coloring. For convenience, we
include the definition of a parameterized decision problem below (see [11] for background in
parameterized complexity).
▶Definition 1 ((Parameterized) decision problem). A parameterized problem is a language
L ⊆Σ∗× N. Given an instance I = (x, k), we call k the parameter. The kth slice Πk of a
parameterized problem Π is a language L ⊆Σ∗with the property that (x, k) ∈Π ⇐⇒x ∈Πk.
Related work
Many works in LFPT are directly relevant to the problems considered
here [1, 16, 29, 33, 34, 36]; further examples include [25, 21, 24, 30].
Many problems
that cannot be solved in linear time can be solved in almost linear time, for example, in
O(n · polylog(n)). This is not only true of Sorting but also, e.g., for deciding first-order
properties in graph classes with locally bounded expansion [12]. Note that in our context,
since log(n) dominates k, f(k) + n · polylog(n) and f(k) · n · polylog(n) define the same class.

Technical Foundations for TLFPT
We start our technical contributions by defining a class of algorithms and a related class of
problems, and some important foundational notions for dealing with these classes.
▶Definition 2 (TLFPT). An algorithm is Truly Linear Fixed Parameter Tractable (TLFPT)
if, on input (I, k) with |I| = n, it runs in time O(n) + f(k). TLFPT denotes the class of all
parameterized problems admitting a TLFPT algorithm.
▶Definition 3 (TLFPT-reduction). Let Π and Γ be two parameterized decision problems, a
map r : Π →Γ is a TLFPT-reduction if there exists a function f such that:
r(x, k) terminates in time O(|x| + f(k)) for each (x, k) in the set of instances of Π, and
returns an instance (x′, k′) of Γ,
(x, k) ∈YES(Π) ⇐⇒(x′, k′) ∈YES(Γ), and
k′ ≤f(k).
We say that Π is TLFPT-reducible to Γ if there exists a TLFPT-reduction from Π to Γ.
In the next lemma, we show that TLFPT-reductions are transitive, and that reducing to
a destination problem that is in TLFPT implies that that source problem is also in TLFPT.
▶Lemma 4. TLPFT-reductions are transitive. In addition, if parameterized problem Π is
TLFPT-reducible to parameterized problem Γ and Γ ∈TLFPT, then Π ∈TLFPT.
Proof. First we show transitivity, suppose a parameterized problem Π is TLPFT-reducible
to parameterized problem Γ and Γ is TLFPT-reducible to parameterized problem Σ; we
will show that Π is TLFPT-reducible to Σ. Let r be a TLFPT-reduction from Π to Γ
and q be a TLFPT-reduction from Γ to Σ. Then we define a function s : Π →Σ by
s((x, k)) = q(r((x, k))) and show that our conditions hold. Let f be a function such that:
r((x, k)) terminates in time O(|x| + f(k)) and has output (x′, k′) satisfying k′ ≤f(k),

The word-RAM architecture
Results in the truly linear paradigm are highly sensitive to
the choice of model. We use the word-RAM model [18, 19], as it is the canonical model for the
analysis and design of fine-grained algorithms [14, 22]. More limited models of computation
in this context are neither as widely studied nor as practically relevant, since they cannot
necessarily support even depth-first search in linear time. Briefly, word-RAM is a unit-cost
RAM with the additional property that the maximum number of bits stored in any single
register – called the word size – is log n, where n is length of the input. Thus word-RAM
prohibits unbounded parallelism wherein constant-time operations are applied to words of
length polynomial in the size of the input.
On the other hand, “word-level parallelism” may be exploited [23].
Such parallelism includes, for example, taking the bitwise AND of two bitstrings each of
length ℓin time O(1) rather than time O(ℓ). A more powerful example: let f : {0, 1}ℓ1 →
{0, 1}ℓ2 be some function stored as a lookup table with 2ℓ1 entries, with ℓ1, ℓ2 at most than
the word size of our RAM. Then for any x ∈{0, 1}ℓ1 we may retrieve f(x) in O(1) time.
As is standard in parameterized complexity, we can assume that n is arbitrarily larger
than k, and in particular that log n > f(k) for any function f (this statement is made
rigorous in the appendix). This allows us, for example, to represent a graph of size k with a
single word, and operate on it in constant time using bitwise operations.
▶Lemma 5. When considering the (non-)existence of a word-RAM algorithm for a para-
meterized problem Π, it suffices to fix any function f and consider only those instances (x, k)
of Π satisfying that |x| > 2f(k).
Proof. We shall show that for any function f, a word-RAM algorithm A deciding problem Π
in TLFPT time exists if and only if there exists a RAM algorithm B with max(f(k), log n)-bit

if f(k) ≤log n, then B can directly be run on a word-RAM with no changes to its behavior
or runtime.

The size of n is bounded by a function of k (specifically, n < 2f(k)), so the runtime of the

Linear Time Kernelization
Kernelization has seen extensive use within parameterized
algorithmics, and is central to many of our results. Here we define linear time kernelization
and then argue that it is fundamentally coupled with the class of TLFPT problems.
We say that a parameterized problem Π admits a linear time kernelization if there exists
a function g and a self-reduction π: (x, k) 7→(x′, k′), computable in time linear in |x| + k,
such that: 1. (x, k) is a yes-instance if and only if (x′, k′) is a yes-instance; and 2. |x′| ≤g(k)
and k′ ≤k. If there is a self-reduction respecting properties (1) and (2) but with runtime
O(|x| + f(k)) for some superlinear f then we refer to “TLFPT time kernelization” or say
that a kernel for Π is computable in time O(|x| + f(k)). There is some variability in the
definition of kernelization. By default (e.g., in [10]), a kernelization’s runtime is polynomial
in |x|+k. In this sense, TLFPT time kernelizations are not strictly kernelizations themselves,
though as we shall see in Corollary 10, any problem admitting a TLFPT time kernelization
also admits a linear-time kernelization.
▶Theorem 9. A nontrivial parameterized problem Π admits a linear time kernelization if
and only if it is in TLFPT.
Proof. A kernelization of an instance (x, k) of parameterized problem Π, linear in |x| + k,
runs in time c|x| + ck for some constant c and yields a kernel of size at most some function
g(k). Overall, the time complexity to solve (x, k) is c|x| + ck + g′(k) for some g′: applying
the kernelization takes time c|x| + ck and solving the resulting kernel takes time f(|x′|, k′)
for some f.
Conversely, we choose fixed yes- and no-instances to reduce to (they exist, since Π is
nontrivial). There must exist an algorithm A with complexity c|x| + f(k) which witnesses
TLFPT membership. We distinguish between two cases.
If |x| > f(k) then the runtime of A((x, k)) is c|x| + f(k) ≤(c + 1)|x| which is linear in |x|
(and in |x| + k), so we can simply solve the instance and then map it to a constant-size
yes- or no-instance, as appropriate.
Otherwise, |x| ≤f(k) and we map (x, k) to itself, which satisfies the definition of a linear
time kernelization (by letting g = f).
We note that this does not require that f be computable: we can check whether |x| > f(k)
by simply running A((x, k)) for (c + 1)|x| steps. If it has terminated then we are in the first
case (and return a constant-size yes- or no-instance), and otherwise we are in the second
case (and return (x, k) directly).

TLFPT via Degree-Based Kernelization: a Warmup
In this section we describe a general approach to proving membership in TLFPT. In particular,
we show that if a problem is degree reducible and has bounded matching number then we can
construct a kernel for the problem in TLFPT time.
We illustrate this method using Vertex Cover, which is both one of the classic NP-
complete combinatorial problems [27], and also (with parameter k) one of the prototypical
FPT problems [11]. A linear kernelization algorithm for this problem has been previously
described [3] and is known to be linear [10]. Here we use the problem to demonstrate our
more general method.
Vertex Cover
Input: graph G = (V, E), integer k.
Question: does there exist a set S ⊆V with |S| ≤k such that for every (u, v) ∈E,
{u, v} ∩S ̸= ∅?
A common observation
is that any vertex with degree at least δ must be contained in any vertex cover of size at
most δ −1. That is, if a graph G contains some vertex cover C of size at most k, then C
necessarily contains every vertex of G with degree at least k + 1. Thus, for any vertex v with
degree at least k + 1, (G, k) is a yes-instance of Vertex Cover if and only if the same holds
for (G −v, k −1). Further, if G contains some isolated vertex v, then (G, k) is a yes-instance
of Vertex Cover if and only if the same holds for (G −v, k). We call a problem with these
properties degree reducible as we formally define below.
▶Definition 11. Let Π be some parameterized graph problem. We say that Π is d-degree
reducible for some function d : N →N, if for any instance (G, k) of Π and any vertex v of G
the following holds:
if |N(v)| > d(k), then (G, k) ∈Y ES(Π) if and only if (G −v, k −1) ∈Y ES(Π); and
if |N(v)| = 0, (G, k) ∈Y ES(Π) if and only if (G −v, k) ∈Y ES(Π).
As suggested above, Vertex Cover is d-degree reducible where d is the identity function.
▶Lemma 12. Vertex Cover is d-degree reducible when d(k) = k for each k ∈N.
Proof. We show d-degree reducibility of Vertex Cover by showing each condition in
Definition 11 holds.
Let (G, k) be an instance of Vertex Cover. Suppose G contains some vertex v with
degree at least k + 1. We claim that G contains a vertex cover of size at most k, if and only
if G −v contains a vertex cover of size at most k −1.
Suppose that there is some vertex cover C ⊆V of size at most k. We claim that v ∈C.
Otherwise, given every edge in G has at least one endpoint in C, N(v) ⊂C. This is a
contradiction as |N(v)| > |C|. It follows that C \ {v} is a vertex cover of G −v and so
(G −v, k −1) must also be a yes-instance of Vertex Cover.
Now let (G −v, k −1) be a yes-instance of Vertex Cover, that is, there is some vertex
cover C ⊆V \ {v} of size at most k −1. It follows that C ∪{v} is a vertex cover of G with
size at most k.
We now consider isolated vertices. Let v be some vertex with degree 0 in G. It follows
that v is not contained in any minimal vertex cover of G and so (G, k) is a yes-instance of
Vertex Cover if and only if (G −v, k) is.

TLFPT kernelization via Matchings and Twins: the Census Method
We now describe a second method for linear time kernelization.
To apply this, the problem must satisfy two properties. The first is bounded matching
number (c.f. Definition 14) which implies that for any non-trivial instance we can compute
a vertex cover of bounded size in linear time. We then construct a census, with respect
to this vertex cover, which characterizes the remaining cloud of vertices according to their
neighborhood, thus describing an equivalence relation over the vertices.
Two vertices are open twins if they are non-adjacent and have the same neighborhood.
That is, a set of open twins must be an independent set. As each equivalence class corresponds
to a set of open twins, the second property that we use, twin reducibility, allows us to bound
the size of each equivalence class and yields a problem kernel.
To illustrate this method, we show that Dominating Set is in TLFPT when paramterized
by vertex cover number. Note that Dominating Set is shown to be W[2]-hard with respect
to the size the dominating set [9]; we instead parameterize by the vertex cover number.

function of k. That is, we take the minimum of some τ(k) and the number of vertices with
that label. We refer to problems where we can truncate the count in the census vector by
some function τ, as τ-twin reducible.
The second property that we require for the census method ensures that we only have to
consider a bounded number of vertices with the same neighborhood. For dominating set, we
need only consider at most c vertices with the same neighborhood. This follows from the fact
that no dominating set of size c can be a strict subset of a set of at least c open twins; either
the remaining vertices are undominated, or all are dominated by a shared neighbor. With
our preprocessing of isolated vertices, this implies there is always at least one twin in the set
which is not in a dominating set of size c. Ensuring that each such vertex is dominated will
also dominate all vertices not included in our bounded census.

Many parameterizations of SAT aim to capture the structural properties of an instance,
particularly the relationships between its variables and clauses. A common way to express
this structure is to encode the formula as a graph and analyze structural parameters of
that graph. Notable examples include the treewidth of the associated graph, as well as the
parameter known as maximum deficiency, as introduced in [17], which is based on matchings
in this graph.
There are several ways to encode a SAT instance as a graph.
Recall that the incidence graph, I(ψ), of ψ is that bipartite graph such that one part
contains the variables of ψ and the other part contains the clauses of ψ. Two vertices are
adjacent to one another in I(ψ) if one of these vertices corresponds to a clause and the other
corresponds to some variable that appears in this clause. We will focus on the incidence
graph and in particular the treedepth of the incidence graph, which we call its incidence
treedepth.
It will be useful to consider the following variant of the incidence graph which we will
call the literal incidence graph. For a CNF formula ψ, the literal incidence graph, IL(ψ), of
ψ is that bipartite graph such that one part contains the clauses of ψ and one part contains
the literals of ψ. That is, for each variable x in ψ, IL(ψ) contains a pair of vertices, one
corresponding to x and the other to ¬x. The following lemma will allow us to work on the
literal incidence graph.

TLFPT via Dynamic Programming: Parameterizing by BFS-Width
In Sections 3–5, we gave TLFPT results via kernelization. We now turn to another workhorse
of parameterized complexity: dynamic programming. It may seem that dynamic programming
algorithms align more naturally with the multiplicative (rather than additive) definition of
FPT. Maintaining a state space bounded by a function of k, a classical algorithm would need
to expend f(k) operations for each of the O(n) parts the decomposition processed, yielding
a runtime on the order of f(k) · O(n). In fact, we show that by carefully considering our
model of computation we can also apply dynamic programming with a TLFPT runtime.
In this section, we parameterize graph problems by BFS-width, a parameter which in some
sense mirrors treedepth: graphs of bounded treedepth have “nice” DFS trees, a structure
we leverage in Section 5, whereas graphs of bounded BFS-width have “nice” BFS trees.
The techniques we apply here could plausibly be lifted to other decompositions (e.g. tree
decompositions), but the question of producing an optimal tree decomposition
in time TLFPT parameterized by treewidth remains open. BFS-width, on the other hand,
has the desirable property that producing a decomposition in linear time is straightforward.
This natural parameter was only recently formally studied from a parameterized perspective
[13].

Discussion and Future Work
Inspired by the realities of modern computational needs, ours is an early exploration of truly
linear FPT. Here distinctions between the additive and multiplicative definitions of FPT
make a difference: LFPT – O(n) · f(k) – strictly contains TLFPT – O(n) + f(k).
Towards being of service to practical computation, we have concentrated on the positive
toolkit of TLFPT. Via appropriate parameterizations and paying attention to the data
structures involved in linear-time algorithmics, we placed many problems in TLFPT including:
SAT, Vertex Cover, Min-Max Matching, (n −k)-Coloring, Diverse Pair of
Matchings, k-Path, k-Vertex Ranking and H-Coloring.
In the case of H-Coloring, where we parameterized by BFS-width and used dynamic
programming, the similarity to parameterizations by tree-width is evident. So we ask:
▶Question 1. Is MSO2 model checking in TLFPT parameterized by BFS-width?
Turning to k-Path, our result is a first step towards being able to integrate TLFPT and
the graph minors machinery; thus we ask:
▶Question 2. For which families {Hi}i∈N is Hi-minor TLFPT parameterized by |Hi|?
Although not our focus here, the negative toolkit is equally important. By diagonalization,
we have shown that there exist parameterized problems in LFPT (and thus in FPT) that
are not in TLFPT. However, still lacking natural TLFPT-hard problems and techniques for
proving TLFPT-hardness, we identify the following direction for further work.
▶Question 3. Which FPT problems are likely not to be in TLFPT?
Bumpus, Downey, Eagling-Vose, Enright, Fellows, Kutner, Larios-Jones, Martin, Rosamond, and Yates

As we mentioned in Section 6, it seems plausible to generalize our dynamic programming
results to classes of “bounded width”. Of course, this would be useful only if we have access
to the corresponding decompositions. We ask:
▶Question 4. Is pathwidth TLFPT parameterized by pathwidth?
▶Question 5. Is treewidth TLFPT parameterized by treewidth?
▶Question 6. Is rankwidth TLFPT parameterized by rankwidth?
There is no shortage of directions for further research. Consider, for example, program-
matic questions such as determining how TLFPT fares in practice, or how to design TLFPT
approximation algorithms. In general, the theory of TLFPT is incipient and we expect its
development to spark healthy conversation between theory and practice.

A
Extended discussion of word-RAM architecture
Because we are interested in truly linear computation, the details of accessing and updating
information about a graph are very important. We also therefore need to be clear on our
model of computation. We use the word-RAM model [18, 19] because it is the canonical
model used for the analysis and design of fine-grained algorithms [22, 14], though this is
sometimes left implicit.
A RAM machine [6] classically consists of a finite program operating on an infinite
sequence of registers, each with the capacity to hold an arbitrary integer. There are two core
versions of the RAM model: one in which reading (or writing) the integer n to a register, or
accessing the nth register, takes constant time (“unit-cost RAM”), and another in which
the same operations take time log |n|. The former is liable to abuse of the constant-time
assumption (e.g. by generating operands of size polynomial in the original input size, each
of which can be stored in a single register), whereas the latter is limited compared to the
actual hardware algorithms are typically executed on, which can indeed access registers (up
to some bound) in constant time.
The word-RAM model [18, 19] is a so-called trans-dichotomous model – it aims to bridge
the gap between these two paradigms. The operation set (and the cost of any operation)
is the same as for the unit-cost RAM model, but with the additional constraint that each
register can only hold words consisting of up to b bits, with b = log2⌈N⌉where N is the size
of the program input. Thus, arithmetic operations with operands of size at most that of the
input can be performed in constant time, but (consistent with the reality that computers
have fixed word length) the model forbids “computations that achieve hidden parallelism by
doing operations on ‘long words’.” [19]
Importantly, we will see that it is possible for a word to be long with respect to a parameter
k while still being sublinear with respect to the overall input size n. This reflects a sound
assumption: that if k is sufficiently small with respect to n, the (real-world) architecture
which enables constant-time operations with operands of size n can be leveraged to operate
on “long words” of size f(k) ≤n.
Recall that w = log n and that we may assume n >> k (if n is upper-bounded by any
function of k then we can simply apply an arbitrary algorithm which decides the problem at
hand; the runtime of this algorithm on such an input is then also bounded by a function of
k).
